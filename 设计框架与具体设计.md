# 魔法学院大挑战 — 设计框架与具体设计汇总

## 一、产品概述

**魔法学院大挑战** 是一款基于 **Taro 4.x + React 18 + TypeScript** 构建的跨平台消除匹配小游戏。玩家通过交换相邻魔法元素完成消除获取分数，配合课程体系与角色养成机制，在限时挑战中追求更高评价。

| 项目 | 内容 |
|------|------|
| 框架 | Taro 4.2.0 (Vite 编译器) |
| UI 方案 | React 18 + TypeScript |
| 样式方案 | CSS Modules（`*.module.scss`） |
| 状态管理 | React Hooks（useState / useEffect / useCallback） |
| 目标平台 | 微信 / 支付宝 / 抖音小程序 + H5 |
| 设计稿基准 | 750rpx |

### 3.3 数据流架构

```
玩家操作 → GameBoard(棋盘) → gameLogic(匹配消除) → 得分/连击更新
     ↓                                                    ↓
useState 状态管理 ← ← ← ← ← ← ← ← ← ← ← ← ScoreDisplay/ComboDisplay
     ↓
GamePage(游戏页) → 游戏结束 → ResultPage(结果页) → 参数传递(URL params)
```

## 四、功能模块设计

### 4.1 课程系统

4 门课程，每门 4 个关卡：

| 课程 | 解锁 | 元素组合 |
|------|------|----------|
| 火焰与冰霜 | ✅ 已解锁 | 🔥 + ❄️ |
| 自然与生命 | ✅ 已解锁 | 🌿 + ✨ |
| 符文与召唤 | ✅ 已解锁 | 🔮 + 👻 |
| 时间与空间 | 🔒 锁定 | ⏳ + 🌀 |

### 4.2 角色系统

| 角色 | 解锁 | 技能加成 |
|------|------|----------|
| 火焰巫师 | ✅ 已解锁 | 火焰元素 +20% 得分 |
| 冰霜法师 | ✅ 已解锁 | 冰霜元素 +20% 得分 |
| 自然德鲁伊 | ✅ 已解锁 | 自然元素 +20% 得分 |
| 符文学者 | 🔒 锁定 | 符文元素 +20% 得分 |

### 4.3 消除玩法

- **棋盘**：9×9 网格
- **操作**：点击选中 → 点击相邻交换（上下左右）
- **匹配规则**：横向或纵向 3 个及以上同元素自动消除
- **连锁消除**：消除后掉落填充，若再次匹配则继续消除（Combo）
- **得分计算**：
  - 基础分 = 消除数 × 10
  - Combo 倍率：×1（2连）、×1.5（3连）、×2（5连以上）
  - 角色技能加成：匹配元素与角色专精匹配时额外加成

### 4.4 特殊消除效果

| 消除类型 | 道具效果 |
|----------|----------|
| 四消 | 生成条纹道具，消除整行或整列 |
| 十字消 | 生成炸弹道具，消除 3×3 区域 |
| 五消 | 生成彩虹道具，消除所有同类型元素 |

### 4.5 关卡机制

- **限时挑战**：每关 50~60 秒
- **目标分数**：5000~15000 分
- **评价等级**：S（≥200%）/ A（≥150%）/ B（≥100%）/ C（≥70%）/ D（<70%）

## 六、技术实现要点

### 6.1 框架配置

- **CSS Modules**：在 `mini` 和 `h5` 配置中同时开启，采用 `[name]__[local]___[hash:base64:5]` 命名模式
- **Vite 编译器**：使用 Taro 4.x 的 Vite 编译模式，支持 HMR 热更新
- **设计稿适配**：`designWidth: 750`，`deviceRatio` 覆盖主流设备

### 6.2 游戏引擎核心算法

消除玩法在 `gameLogic.ts` 中实现，核心算法包括：

| 算法 | 说明 |
|------|------|
| `initializeBoard` | 初始化无预匹配的 9×9 棋盘 |
| `isValidMove` | 验证交换是否产生有效匹配 |
| `findMatches` | 扫描横纵方向找出所有 ≥3 连匹配 |
| `classifyMatches` | 分类匹配：普通匹配、四消、十字消、五消 |
| `markMatches` | 标记匹配元素为 `isMatched: true` |
| `createSpecialElements` | 根据匹配类型创建特殊道具（条纹、炸弹、彩虹） |
| `triggerSpecialEffects` | 触发特殊道具效果 |
| `dropElements` | 匹配消除后元素下落填补空位 |
| `fillEmptyCells` | 顶部补充新随机元素 |
| `calculateScore` | 基于消除数、连击数、角色技能计算得分 |

### 6.3 状态管理方案

游戏状态通过 React Hooks 管理，使用 `useRef` 解决闭包问题：

```typescript
// 状态变量
const [score, setScore] = useState(0);
const [timeLeft, setTimeLeft] = useState(60);
const [maxCombo, setMaxCombo] = useState(0);
const [course, setCourse] = useState<Course | null>(null);
const [character, setCharacter] = useState<Character | null>(null);
const [levelId, setLevelId] = useState(1);

// 使用 ref 存储最新状态，解决 setInterval 中的闭包问题
const gameStateRef = useRef({ score, timeLeft, maxCombo, course, character, levelId });

// 同步状态到 ref
useEffect(() => {
  gameStateRef.current = { score, timeLeft, maxCombo, course, character, levelId };
}, [score, timeLeft, maxCombo, course, character, levelId]);
```
